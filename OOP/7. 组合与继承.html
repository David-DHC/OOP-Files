    2. 组合所得对象的构造与析构次序：

      1. 先完成子对象构造，再完成当前对象构造
      2. 子对象构造的次序仅由在类中声明的次序所决定
      3. 析构函数的次序与构造函数相反

     

    1. 如果组合类需要调用子类的拷贝构造函数（或者重载的赋值号），那么除非有显式定义好的函数，否则一律使用位拷贝。

    4. 多重继承：一个派生类继承多个基类的继承方式。

      示例代码：

      xxxxxxxxxx
      3
      1
      class InputFile{}; 
      2
      class OutputFile{}; 
      3
      class IOFile: public InputFile, public OutputFile {}; //IOFile类继承了InputFile和OutputFile两个类型

      应当慎用多重继承，因其容易造成两个问题：

      数据冗余：仍以上述示例代码为例，若InputFile和OutputFile两类都继承了File类，那么被继承数据将在IOFile中双份存储，造成空间浪费。

      二义性：仍以上述示例代码为例，若InputFile和OutputFile两类中有同名数据a，那么IOFile.a这个数据就具有二义性，造成编译错误。

     

    1. 若希望定义某类不可被继承，则需要在类的声明中使用final关键字。

      示例代码：class A final {};

      示例注释：接下来的任何类都不能以A为基类进行继承。
